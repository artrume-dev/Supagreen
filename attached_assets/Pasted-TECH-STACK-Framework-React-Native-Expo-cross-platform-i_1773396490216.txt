               OR Next.js 14 (App Router) for web-first MVP
Backend:       Node.js + Express (or Next.js API routes)

CORE AI RECIPE GENERATION

System prompt for each daily generation:

  "You are a certified nutritionist and chef. Generate exactly 
   3 complete recipes (breakfast, lunch, dinner) for a user 
   with the following profile:
   - Diet: {diet_type}
   - Allergies: {allergies}
   - Goal: {health_goal}
   - Skill level: {skill_level}
   - Location: {city}, {country}
   - Season: {current_season}
   - Daily calorie target: {calories} kcal
   
   STRICT RULES:
   1. Every single ingredient must be a 100% whole, unprocessed food.
      No protein powders, artificial sweeteners, processed snacks,
      refined sugar, white flour, seed oils (canola, soybean, sunflower),
      or ultra-processed ingredients.
   2. Ingredients must be SEASONALLY AVAILABLE in {country} 
      during {current_season}.
   3. Provide macros (protein g, carbs g, fat g, calories) per serving.
   4. Prep time must match skill level.
   5. Each recipe must serve the goal: e.g. high-protein for muscle 
      building, anti-inflammatory foods for inflammation goals.
   6. Format response as valid JSON matching this schema:
      {
        recipes: [
          {
            meal: 'breakfast' | 'lunch' | 'dinner',
            title: string,
            emoji: string,
            prepTime: number (minutes),
            servings: number,
            healthScore: number (1-10),
            goalAlignment: string (one sentence),
            macros: { calories, protein, carbs, fat },
            ingredients: [
              { name, amount, unit, isKeyIngredient: boolean }
            ],
            steps: [string],
            healthBenefits: [string] (max 3 bullet points),
            swapSuggestion: string (one easy ingredient swap)
          }
        ]
      }
   Return ONLY valid JSON. No markdown, no preamble."

Cache generated recipes in Supabase with the date + user_id as key.
Regenerate only if: user requests it, date changes, or preferences update.
Allow user to "regenerate one meal" (costs 1 API call) up to 3x/day.

STORE & INGREDIENT AVAILABILITY FEATURE
─────────────────────────────────────────────────────────────
After recipe generation:
1. Extract all ingredients from today's 3 recipes into a unified list
2. Call Google Places API with:
   - query: "supermarket" OR "health food store" OR "grocery store"
   - location: user lat/lng
   - radius: 3000m (adjustable in settings)
   - type: supermarket
3. Return top 5 nearest stores with:
   - Store name, chain logo (if available), distance, opening hours
   - Google Maps deeplink for directions
   - "Tap to navigate" CTA
4. Display a combined shopping list grouped by store section:
   (Produce / Protein / Grains / Dairy alternatives / Pantry)
5. Allow user to check off items as they shop
6. "Share shopping list" button → exports as plain text or saves to 
   Apple Reminders / Google Tasks via deep link

ADDITIONAL AI FEATURES (progressive enhancement)

1. NutriChat — floating AI assistant button on every screen
   "Ask me anything about today's recipes or nutrition"
   Uses Claude with conversation history + user profile as context

2. Ingredient scanner — camera scan a food item → Claude identifies 
   it and tells you if it fits their diet + how to use it

3. Fridge mode — user types or speaks what's in their fridge →
   Claude generates a compliant recipe from those exact ingredients

4. Meal swap — tap any ingredient → Claude suggests 3 whole-food 
   swaps with nutritional comparison

5. Weekly meal prep guide — Sunday feature: Claude generates a prep 
   plan to batch-cook the week's meals efficiently
